# HealthIQ AI — v5.3 Depth Track (Document Only)

This document defines modelling primitives for the v5.3 depth track.
It is planning-only and does not authorize implementation in v5.2.

## 1. Multi-Marker State and Trajectory Inference

- Purpose: infer deterministic biological state signatures from grouped biomarkers and derived markers, then track state direction across snapshots.
- Inputs required (existing contracts): `insight_graph.biomarker_nodes`, `insight_graph.relationships`, `insight_graph.biomarker_context`, and replay-linked prior snapshots.
- Deterministic constraints: no probabilistic branching; fixed rule precedence; stable ordering and code-only outputs.
- Replay stamp impacts: add a dedicated state-engine version/hash to replay manifest and stamp output schema version in InsightGraph.

## 2. Interaction Precedence and Conflict Resolution

- Purpose: resolve deterministic conflicts when signals disagree (for example, marker status vs relationship signal vs context code).
- Inputs required (existing contracts): `biomarker_nodes.status/score`, `relationships.triggered/evidence`, `biomarker_context.reason_codes` and `relationship_codes`.
- Deterministic constraints: explicit precedence table, no fuzzy tie-breaking, deterministic arbitration evidence codes.
- Replay stamp impacts: version/hash stamp for precedence policy and conflict-rule schema.

## 3. Uncertainty Decomposition Beyond Completeness

- Purpose: decompose confidence into deterministic components (coverage, signal coherence, conflict burden) instead of completeness-only confidence.
- Inputs required (existing contracts): `confidence` model fields, cluster completeness, relationship trigger density, conflict outputs.
- Deterministic constraints: fixed formulas and bounded transforms; no learned/adaptive scoring in v5.3 baseline.
- Replay stamp impacts: confidence-model version increment and deterministic hash coverage for decomposition policy.

## 4. Longitudinal State Transition Classification

- Purpose: classify transition patterns (improving, worsening, unstable, persistent) using deterministic deltas and state transitions.
- Inputs required (existing contracts): replay-linked prior `insight_graph` snapshots, current `biomarker_context`, current `relationships`, and confidence context.
- Deterministic constraints: explicit delta thresholds, canonical transition classes, no heuristic narrative shortcuts.
- Replay stamp impacts: transition-policy version/hash and per-analysis transition schema stamp in replay manifest.

## 5. Outcome Calibration Layer (Deterministic Calibration Registry)

- Purpose: calibrate deterministic inference outputs against observed outcome labels using governed, versioned calibration maps.
- Inputs required (existing contracts): normalized inference codes from InsightGraph, replay-linked longitudinal windows, externally curated outcome labels.
- Deterministic constraints: immutable calibration releases, no online learning in runtime, strict version pinning per analysis.
- Replay stamp impacts: calibration-registry version/hash in replay manifest and output contract version notes for calibrated outputs.
