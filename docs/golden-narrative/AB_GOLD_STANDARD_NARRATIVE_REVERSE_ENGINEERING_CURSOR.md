# HealthIQ AI — Reverse Engineering of AB Gold Standard Narrative

## 1. Executive summary
- The benchmark narrative is **partly supportable today** from the current backend and governed asset model. The repo already has strong deterministic foundations for: canonical biomarker identity, lab-shaped panel inputs, structured signal detection, ranked top findings, root-cause hypotheses, confirmatory tests, clinician summary output, and some bounded reassurance assets such as `balanced_systems_v1`.
- The strongest existing backend support is around the **lead homocysteine signal stack itself**: current/past AB fixtures, governed signal libraries, root-cause hypotheses, confirmatory-test registry, deterministic report compilation, and questionnaire/lifestyle intake all exist.
- The largest deterministic gaps are not raw data gaps. They are **asset-shape gaps**: there is no governed **methylation-first phenotype/IDL asset**, no deterministic **longitudinal prose compiler**, no governed **lifestyle-to-pathway interpretation join** for alcohol/hydration/weight-loss context, no deterministic **cross-system reassurance synthesizer** at the benchmark's level of nuance, and no governed asset for the benchmark's **protective lipid-context narrative**.

## 4. Existing backend strengths
- **Canonical biomarker governance is strong.** The repo already has a clear governed biomarker registry and canonical IDs for the main AB markers used in the benchmark, including homocysteine, CBC indices, B12/active B12/folate, CRP, renal markers, liver markers, lipids, ApoB/ApoA1, Lp(a), ferritin, transferrin, thyroid markers, and testosterone. Primary authority: `backend/ssot/biomarkers.yaml`.
- **The AB data foundation is strong.** The current AB panel fixture is rich, lab-shaped, and contains per-analyte lab ranges. There is also a plausible prior-panel companion fixture matching the benchmark's longitudinal comparisons. Primary files: `backend/tests/fixtures/panels/ab_full_panel_with_ranges.json` and `backend/tests/fixtures/panels/ab_full_panel_with_ranges_test0.json`.
- **Ranking and structured reporting are already deterministic.** `report_v1`, `clinician_report_v1`, and the report compiler can already rank concerns, surface chains, present confirmatory tests, and provide a structured clinician-facing summary. Primary files: `backend/core/contracts/report_v1.py`, `backend/core/contracts/clinician_report_v1.py`, and `backend/core/analytics/report_compiler_v1.py`.
- **The homocysteine root-cause stack is already materially useful.** There is governed support for B12-associated, folate-associated, inflammatory-context, and renal-clearance-context hypotheses, plus confirmatory tests such as MMA and repeat homocysteine. Primary files: `knowledge_bus/root_cause/hypotheses/hcy_hypotheses_v1.yaml` and `knowledge_bus/registries/confirmatory_tests_v1.yaml`.
- **The system already has bounded positive-context infrastructure.** `balanced_systems_v1`, system states, and Layer C features show that deterministic "what is working well" scaffolding exists, even if it is not yet benchmark-grade. Primary files: `backend/core/analytics/balanced_systems_presentation_v1.py`, `backend/core/contracts/insight_graph_v1.py`, and `backend/core/analytics/insight_graph_builder.py`.
- **Questionnaire/lifestyle intake exists as governed backend input.** Alcohol, exercise, sleep, stress, fluid intake, and related context are already mappable into backend structures. Primary files: `backend/ssot/questionnaire.json`, `backend/ssot/lifestyle_registry.yaml`, and `backend/core/pipeline/questionnaire_mapper.py`.

## 5. Deterministic asset gaps
- **No methylation-first phenotype/IDL asset.** The current phenotype/IDL stack does not provide a governed `ph_*` record that says "homocysteine-macrocytosis / methylation-efficiency pattern." Existing phenotype framing is more vascular/inflammation-oriented: `knowledge_bus/phenotypes/phenotype_map_v1.yaml` and `knowledge_bus/interpretation_display_layer_v1/idl_records_v1.yaml`.
- **No deterministic cross-system reassurance synthesizer at benchmark depth.** The backend can say some systems are stable, but it cannot yet deterministically produce the benchmark's higher-order move: "this is not broad metabolic deterioration; the background is calm, so the unresolved pathway stands out more clearly."
- **No deterministic lifestyle-to-pathway interpretation join.** Questionnaire facts such as alcohol intake and fluid intake are ingested, but there is no governed asset that turns those into report-safe pathway interpretation for homocysteine/macrocytosis or renal context.
- **No deterministic longitudinal prose layer.** State transitions exist at biomarker level, but the clinician report and IDL do not currently consume them into "since panel 0" narrative sections.
- **No governed protective-lipid-context narrative asset.** Current lipid support is strongest for non-HDL-driven dysfunction and confirmatory tests. The benchmark's more nuanced "mild residual LDL exposure inside an otherwise protective transport profile" is not presently encoded as a deterministic phenotype or summary class.
- **No governed combined vascular synthesis of homocysteine plus lipid context.** The benchmark's "two vascular mechanisms, one buffered and one less resolved" requires a cross-system phenotype/chain that does not currently exist.
- **Confirmatory and follow-up coverage is incomplete for the benchmark.** MMA and repeat homocysteine are covered, but B6/P5P and alcohol-reduction-as-diagnostic-context are not represented as governed deterministic assets.

## 6. Recommended next build implications
- **New compiler asset needed:** deterministic body-overview / panel-posture compiler that can responsibly express "overall reassuring" and "not broad deterioration" from system states, low-burden evidence, and ranking context.
- **New phenotype + IDL asset needed:** a governed methylation / one-carbon / homocysteine-macrocytosis phenotype record with user-safe `why_it_matters`.
- **New longitudinal comparison asset needed:** deterministic report section that consumes `state_transitions` and prior/current panel pairings into bounded trend language.
- **New lifestyle-to-interpretation asset needed:** governed joins for alcohol, hydration, fasting pattern, exercise pattern, and possibly weight change where that context materially changes interpretation.
- **New pathway explainer asset needed:** reusable deterministic biology copy layer for homocysteine/methylation and for residual LDL/protective lipid transport interpretation.
- **New cross-system phenotype needed:** a governed interaction asset that can combine homocysteine-related vascular concern with lipid transport context.
- **Existing governed asset only needs surfacing:** ranked lead concern, clinician summary structure, root-cause hypotheses, confirmatory tests, and balanced-systems reassurance already exist and should be reused rather than rebuilt.
- **Expand confirmatory-test and action assets:** add governed support for B6/transsulfuration clarification and pathway-specific lifestyle challenge logic if those are desired as deterministic next steps.
